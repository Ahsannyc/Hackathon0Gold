# 🚀 Quick Start Setup - What You Need to Run the Project

## Answer: Will It Run?

**YES ✅** - The project will run with just basic setup.

## Minimum Setup (5 minutes) - Project Will Run

### Step 1: Create `.env` file
```bash
# Copy the template
cp .env.example .env

# Edit .env and add only these REQUIRED values:
POSTGRES_PASSWORD=choose_a_secure_password_123
POSTGRES_USER=odoo
POSTGRES_DB=hackathon0_business
ODOO_MASTER_PASSWORD=choose_a_secure_master_password
```

### Step 2: Start Docker
```bash
docker-compose up -d
```

### Step 3: Verify It's Running
```bash
# Backend API (should show "Swagger UI")
curl http://localhost:8000

# Frontend (should show web interface)
curl http://localhost:3000

# Database (should be running)
docker ps | grep postgres
```

**Result:** ✅ Project is running on localhost:3000

## Common Issues & Solutions

### Error: "POSTGRES_PASSWORD is required"
**Fix:** Make sure `.env` exists and has `POSTGRES_PASSWORD=something`

### Error: "credentials.json not found"
**Fix:** Either run `python authenticate_gmail.py` OR just skip Gmail (it's optional)

### Error: "Connection refused on port 5432"
**Fix:** Wait 30 seconds for Docker container to start, then try again

### Port Already in Use
```bash
# If port 3000 or 8000 is in use:
docker-compose down
# Then restart
docker-compose up -d
```
